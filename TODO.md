# Fact-Finding Techniques

To identify the requirements of the Teaching Assistant (TA) Recruitment System, the team plans to apply several fact-finding techniques. These techniques aim to understand the current recruitment workflow, identify existing problems, and gather user expectations for the proposed system.

## 1. Stakeholder Identification

The first step is to identify the main stakeholders who interact with the TA recruitment process. Understanding the needs of these stakeholders will help define the system scope and guide the creation of user stories in the product backlog.

The primary stakeholders include:

- **Teaching Assistants (TAs):** Students who apply for assistant positions.
- **Module Organisers (MOs):** Staff members who post jobs and select applicants.
- **Administrators:** Staff responsible for monitoring TA workload and managing recruitment records.

By identifying these stakeholders early, the team can ensure that the system design addresses the needs of all key users.

## 2. Interviews

The team plans to conduct semi-structured interviews with potential users such as TAs and Module Organisers. Interviews will help the team gain detailed insights into the current recruitment process and identify challenges experienced by users.

During the interviews, the team will focus on understanding:

- How the current recruitment process works
- What problems exist in the current recruitment approach
- What features users expect from a digital recruitment platform

The information gathered from interviews will help the team define user stories and identify essential system functions.

## 3. Document Analysis

The team will analyse existing recruitment materials, such as application forms and Excel spreadsheets currently used to manage TA recruitment.

This analysis will help the team understand the current workflow and identify the types of information required in the system, including:

- Applicant personal information
- CV submissions
- Job descriptions
- Application status

Document analysis will ensure that the new system supports the existing recruitment requirements while improving organisation and efficiency.

## 4. Prototyping and User Feedback

After gathering initial requirements, the team plans to create a low-fidelity prototype to visualise the interface and workflow of the proposed system. The prototype will include key features such as applicant registration, job browsing, and job application.

The prototype will then be presented to potential users to collect feedback on usability and functionality. This feedback will help the team refine system requirements and improve the interface design before implementation begins.
